Создать информационный портал
1. Создаем информационный портал. В нем нужны 2 вида публикаций — новости и статьи. У них общие - заголовок, вступительный текст и полный текст. Отличия — у статей есть автор, у новостей — источник.

Для всех типов записей у нас будет одна таблица в базе данных.

2. Создаем класс Publication и наследующиеся от него классы News и Article. 

Класс Publication в статическом методе-фабрике create($id) по id получает из базы данных нужную запись и в конструктор передает нужные аргументы.

3. Для отображения на главной всех записей. Создадим класс PublicationsViewer, который в конструкторе __constructor($publictionType) будет получать тип записи и сохранять в свое свойство protected $publications объекты нужного типа. (Объекты создаем просто через конструктор, не фабрику). 

Пример $newsViewer = new PublicationsViewer('news'); Конструктор должен сделать запрос в бд и получить все записи у которых тип - news. Дальше создать объекты класса News и сохранить их в свойство $publications.

4. Также PublicationsViewer должен иметь public метод write(), который при помощи цикла foreach создаст строку, с информацией о публикациях (News или Article) и ссылку читать полностью, которая через GET передает id записи. 

При переходе - открывается страница с полным текстом записи.(Тут пригодится метод фабрика который мы создали ранее)
